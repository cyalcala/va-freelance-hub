## 🛠️ Infrastructure Replication (Step-by-Step)

### 1. Database (Turso)
- Create a Turso database.
- Create two critical tables: `opportunities` and `noteslog`.
- **IMPERATIVE**: Ensure the `last_seen_at` column exists in all job-related tables to enable freshness tracking.
- Run `npm run db:generate` followed by `npm run db:migrate`.

### 2. Job Ingestion (Trigger.dev)
- Initialize a Trigger.dev v3 project.
- Deploy the `/jobs` directory.
- **SECURITY**: Mapping secrets (`TURSO_DATABASE_URL`, `TURSO_AUTH_TOKEN`) is mandatory for background execution.
- Config: Use `jobs/trigger.config.ts`.

### 3. Frontend (Vercel)
- Deploy the `apps/frontend` directory to Vercel.
- **INVARIANT**: Set Node.js version to **20.x**.
- **INVARIANT**: Use the `@astrojs/vercel` adapter with `edgeMiddleware: false`.
